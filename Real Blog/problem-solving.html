Reflect on how confident you feel using the problem solving techniques and process:

.Pseudocode .Trying something .Rubber ducky method .Reading error messages .Console.logging .Googling .Asking your peers for help .Asking coaches for help .Improving your process with reflection

If i'm being really honest, when it comes to Pseudocode I am not very confident with the technique. As often I make various misatakes with spacing and case sensitive words. But in terms of formatting I do feel very confident with laying out code correctly for the computer to understand.

Ok so when it comes to trying something I am very confident in using the method. Which mainly involves looking at various ways to solving a code question and using trial and error going through various ways till the correct one becomes apparent.

Unfourntunatly when it comes to the rubber ducky method I am not confident at all. As I often only understand what the entire chunk of code does and only a handful of the code that makes up the complete function/chunk.

With error messages, I find it very helpful as when I am using javascript. Errors appear a lot, and the message can inform me that a value is not defined or the way the code is formatted is inccorect and that once its fixed the code will then work. So my confidence level would probably be 8 out of 10.

Console logging, using a line of code that looks like this: console.log. and then adding a value in a bracket () will then make a word that you put in the code appear in the console part of the devtools feature in a web browser. This can also help with seeing if a return function works, by returing the specific value. I am fairly confident when it comes to using this method.

Finally for the methods of Googling, Asking your peers for help, Asking coaches for help and improving your process with reflection. I can say that I am confident with these methods. So my confidence in this method is very high.
